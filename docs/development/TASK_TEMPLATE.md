# TASK TEMPLATE

Version 1.0

## Purpose

Provide a standard format for implementation tasks so every request is scoped, reviewable, and consistent.

## Template

```md
# TASK [NUMBER] — [TITLE]

## Goal

[Describe the objective in one or two sentences.]

## Requirements

- [Requirement 1]
- [Requirement 2]
- [Requirement 3]

## Constraints

- [Constraint 1]
- [Constraint 2]

## Related Documents

- [Document 1]
- [Document 2]

## Affected Files

- [File 1]
- [File 2]

## Acceptance Criteria

- [ ] TypeScript passes
- [ ] ESLint passes
- [ ] Mobile verified
- [ ] Desktop verified
- [ ] Existing features unaffected
- [ ] Performance maintained
- [ ] Reusable implementation
- [ ] Architecture unchanged
- [ ] Documentation updated if required

## Testing Checklist

- [ ] Run type checking
- [ ] Run linting
- [ ] Verify responsive layout
- [ ] Verify existing behavior
- [ ] Verify documentation updates

## Output Rules

- Output only modified files.
- Do not include unrelated changes.
- Stop after the requested task is complete.
```

## Usage Rules

- Fill in every section before implementation starts.
- Keep the task scope narrow and explicit.
- Add only the files and checks relevant to the task.
